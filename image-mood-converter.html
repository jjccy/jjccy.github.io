Project overview

Project Description

Image Mood Converter is a half month course project for IAT 455: Computational Media. The goal of this project is to explore the use of colour in changing the emotions and mood of a particular image. Through this, we hope to achieve a level of modification such that the supplied images, if they are rather gloomy, with grey skies or generally less appealing colours, can be modified to be warmer and brighter.

We notice that this can be done in Photoshop or other image editing programs, but the problem with those is that the emotional/mood colour shift of those programs require the manual brushing from the user. In this program, we wish to automate it so that the user can colour shift many images in a short period of time.

Process

Inspiration

After we decide on the general goal for this project, I start to do research on these areas. One research paper really inspired me is called Automatic Mood-Transferring between Color Images, written by Yang, Chuan-kai and Peng, Li-Kai. This paper classified an image into 24 colour moods where each image has a different percentage of moods. The paper also introduces an algorithm to shift the input image’s mood to the reference image’s mood by recalculating the RGB value.

We plan to create a program with a GUI on the right side that allows users to upload their own images (first through direct upload and later possibly by grabbing images from the internet URL itself). After successful upload, the user can click a number of buttons, each with its own emotional or mood colours where the image’s colours will shift accordingly.

My job was mainly on the general layout of the code, include import images, saving output images, mood visualizer and some versions of the mood transfer script. I also work on searching research articles and try to adapt their ideas to our project.

Basic Mechanism

There are only total of mechanisms in this project, Include

  1. Load Image
  2. Save Image
  3. Recognize Image
  4. Mood Transfer

I start to work on with load image and save the image first. This allows user to upload image files to the programme and save generated image once complete. I didn't have too much experience on working with JFilesChooser (A class allow the user to choose files) before. This project provides me with a good chance to understand how it works.

I also found out that this class has many interesting features, including from different layouts to file type. Just with some simple coding, I can limited user to only upload the image files to prevent any internal bug.

Mood Converter

Mood converting is the most important part of this project. While researching, I saw many different types of method to achieve this feature, I decided to adapt some of them into this project.

The first method I tried is separate all colours in the reference image into 24 sections by RGB. Choose the most colour section and multiplied with the input image. This method only works when the reference image is dominated by similar color so that is a no. I also tried to separate colours by HSV's Hue value, also separate input image into small sections according to its hue, modified the image according to its section and reference image's colour sector. This method also didn't work out and creates huge amounts of artifacts that we don't want.

Most of the approaches we tried have some issues here and there, so we decide to modified our first method to able to fit more situations. Mood transfer is not a simple project I thought it is. it requires complex algorithms to makes it looks right. At the last minute, I found out some better solutions such as Exact Histogram Match, but we don't have time to learn it. The outcome is not perfect, but ideal.
